## 精修内容

假设你已经有了一个无风险利率，并且估算出了一个股权风险溢价，那么你就具备了估算平均风险投资预期回报所需的基本要素。但如果你的投资并非平均风险呢？你就需要一个相对风险的衡量标准，来判断一只股票的风险是平均风险股票的两倍还是仅为其一半。这基本上就是 Beta 试图解决的问题。Beta 带有大量的理论包袱，因为它源于金融学中那些并非所有人都认同的基础风险回报模型。在本节课中，我将介绍传统 Beta 的运作方式以及如何更好地估算它们，同时我也会提供传统 Beta 的替代方案，这些方案同样旨在衡量一项投资的相对风险。

在之前的两节课中，我们已经涵盖了贴现率模型或股权成本模型的前两个要素：一是无风险利率，其确定可能需要一些工作；二是股权风险溢价，其确定需要一些思考。本节课，我将重点讨论这个过程的第三个输入项。如果你已经有了无风险利率和股权风险溢价，你就有了估算一家平均风险公司股权成本的要素。但如果你的公司并非平均风险呢？本节课的核心任务就是讨论如何衡量相对风险，即如何判断一家公司的风险是平均水平的 1.5 倍还是 0.5 倍。

我将使用 Beta 作为贯穿整个过程的工具，但我想先介绍所有可以用来思考相对风险的不同方法。有些方法源于现代投资组合理论，例如单一 Beta 或多 Beta 模型，它们为整个过程带来了严谨性。你也可以只看基于股价的方法，比如比较某只股票的标准差与市场平均标准差，但这仍然依赖于市场价格。此外，还可以使用会计指标，如会计比率或盈利波动性。但最终，我希望从这个过程中得到的是一个相对风险的度量。如果你不喜欢 Beta，不要放弃相对风险这个概念，而是去寻找一种不同的衡量方式，我们仍然可以在同一框架下讨论。

我将从资本资产定价模型（CAPM）的 Beta 开始，因为我们所知的估值知识大多仍建立在 CAPM Beta 之上。在 CAPM 模型中，所有对市场风险或宏观经济风险的敞口都由一个数字——Beta 来捕捉。如果你上过金融课，或者仅仅是浏览过金融教材，我们都被教导用完全相同的方式来估算 Beta：对股票回报与市场指数回报进行回归分析，回归线的斜率就是 Beta。

这听起来应该很简单，对吧？让我们来试试。我拿巴西航空工业公司（Embraer）的数据跑了一次回归，得到了一个 Beta。这应该就是我估值中使用的 Beta，对吗？但当我再次对 Embraer 进行另一次回归时，却得到了一个截然不同的 Beta。我想传达的第一个信息是：当你谈论回归 Beta 时，你得到的结果取决于你如何设置回归。具体来说，它取决于你选择的回归周期、使用的市场指数，以及你使用的是日回报率、周回报率、月回报率还是年回报率。

回归 Beta 只是思考 Beta 的一种方式，而且不是一种很好的方式。它为你提供了一个关于公司 Beta 的统计答案。需要记住的是，在许多回归分析中，你得到的系数（也就是 Beta）都伴随着一个标准误。换句话说，这个结果可能是错的。回归输出实际上会告诉你它可能错到什么程度，这个数字就是 Beta 的标准误。举个例子，美国公司 Beta 估算值的中位数标准误约为 0.2。这意味着什么呢？如果我告诉你，通过回归分析得出可口可乐的 Beta 是 1.10，标准误是 0.2，那么我实际上是在说，可口可乐的真实 Beta 可能低至 0.7（1.10 - 2 * 0.2），也可能高至 1.5（1.10 + 2 * 0.2）。Beta 估算值是一个充满噪音的估算，尤其是当我们只使用单一回归和单一时间段的数据时，噪音可能会淹没你的估算。

你可能会说，如果我的回归看起来非常好呢？请务必小心，最差的 Beta 估算往往来自那些看起来最好的回归。我最喜欢的一个例子是诺基亚（Nokia）。这是我很久以前为诺基亚打印的一张 Beta 页面。诺基亚是一家芬兰公司，当我从我使用的服务（彭博社）请求 Beta 时，就出现了这个页面。从表面上看，这是一个非常漂亮的回归，R-squared 很高，标准误很低。但这是具有欺骗性的，因为这是诺基亚对赫尔辛基证券交易所指数的回归。你知道在这次回归时，诺基亚在该交易所的权重是多少吗？大约 80%。所以，这本质上是诺基亚对诺基亚的回归，它完全没有告诉你，对于一个投资者来说，将诺基亚加入投资组合会增加多少风险。

因此，我希望大家在思考 Beta 时，不要再指望通过单次回归得出一个可以使用的数字。相反，我们应该从宏观层面思考。一个公司的 Beta 并非来自回归分析，而是来自该公司做出的三个选择。首先，它从事什么样的业务？一般规则是，公司的产品或服务越具有“非必需性”（discretionary），其 Beta 就越高。如果你的客户可以没有你的产品，可以延迟或推迟购买，那么你的 Beta 就应该比那些生产绝对必需品的公司要高。例如，一家杂货店的 Beta 应该低于蒂芙尼（Tiffany）。

其次，公司的成本结构是怎样的？固定成本在总成本中所占的比例越大，公司的 Beta 就越高。为什么？因为如果你有大量的固定成本，好光景会变得极好，而坏光景则会变得极糟，一切都被放大了。因此，如果你处在一个高固定成本的行业，我预计你的 Beta 会比低固定成本行业的公司更高。

第三个也是最后一个因素是财务杠杆。当你借钱时，你就创造了一项在借钱之前并不存在的固定成本——利息支出。利息支出使得好光景对股东来说更好，坏光景则更糟，它再次放大了风险。因此，你借的钱越多，你公司的 Beta 就会越高。所以，当我坐下来为一家公司估值时，我不会去看回归页面，而是从基本面开始：这家公司做什么？根据它的业务，我期望的 Beta 是多少？它的成本结构如何？它借了多少钱？回答这些问题，比简单地将回归线的斜率作为 Beta，能让我对公司的真实 Beta 有更深刻的理解。

我将以这些基本面因素为基础，提出一种替代回归 Beta 的方法，我称之为“自下而上 Beta”（Bottom-up Betas）。不要被这个术语吓到，它的逻辑很简单。假设你是一家公司，想让我估算你的 Beta。我会先问你：你从事哪些业务？假设你告诉我，你同时涉足钢铁和化工业务。我会去找到所有上市的钢铁公司和化工公司。因为它们是上市公司，我可以查到它们的回归 Beta。然后，我会计算出钢铁公司和化工公司的平均 Beta。我知道这些 Beta 会受到债务水平的影响，所以我需要进行清理，这个过程叫做“去杠杆化”（unlevering the beta）。听起来很复杂，但其实我只是在剔除债务的影响，最终得到一个纯粹的、只反映业务本身的“无杠杆 Beta”（unlevered beta）。

接下来，我会问你，你的价值有多少来自钢铁业务，多少来自化工业务？你可能无法直接告诉我价值的构成，但你可以告诉我每个业务的收入。我会用这些收入作为权重，计算出一个加权平均的无杠杆 Beta，这个 Beta 反映了你公司所从事业务的整体风险。最后，我会问你公司的债务和股权情况。你可以给我实际的负债权益比，也可以给我一个目标比率。基于你公司的业务构成和所选择的杠杆水平，我将最终计算出你公司的股权 Beta。

你可能会问，我既然不喜欢回归 Beta，为什么还要用它？我找来的那些钢铁公司和化工公司的 Beta，不都是回归 Beta 吗？是的，我所做的只是用成百上千个回归 Beta 的平均值，来代替你公司那一个充满噪音的回归 Beta。这背后的逻辑是统计学中的大数定律。简单来说，你可以拿 100 个糟糕的 Beta，把它们平均一下，平均值就会神奇地变得非常精确。这是因为，当我取平均时，那些被高估的 Beta 和被低估的 Beta 会相互抵消，我的错误在平均过程中被消除了。这是自下而上 Beta 最大的卖点：因为它是一个跨越众多公司的平均值，所以它比任何单一的回归 Beta 都要精确得多。

自下而上 Beta 还有其他优势。如果你的公司昨天刚进入化工行业，回归 Beta 根本不可能捕捉到这一变化。但如果我使用自下而上的方法，我可以调整业务权重，甚至可以主动地将你计划进入但尚未进入的业务也考虑进来。最后，自下而上 Beta 可以用于估算私营企业的 Beta，而回归 Beta 对私营企业则完全无能为力。因此，我非常推崇自下而上 Beta，我已经有近 15 年没有在估值中使用过回归 Beta 了，这足以说明我对此的坚持。

让我们来看一个更复杂的案例，一个涉及多个业务部门的公司——德国软件公司 SAP。SAP 将自己划分为三个业务，我将其简化为两个：软件和咨询。我找到了上市的软件公司和咨询公司，并计算出了这两个行业的无杠杆 Beta。接下来的一步是估算 SAP 有多少价值来自这两个业务。通过查看 SAP 的财务报表，我可以得到每个业务的收入。如果时间紧迫，我可以直接使用收入权重，但这隐含了一个假设：一美元的软件收入和一美元的咨询收入价值相同，而这可能不成立，特别是当利润率不同时。

为了更精确，我增加了一个估算层。我查看了那些可比公司，不仅看了它们的 Beta，还看了它们的收入乘数（市值/收入）。我观察市场给这些公司定价时，是按照两倍、三倍还是五倍的收入来估值的。然后，我将软件行业的平均收入乘数应用于 SAP 的软件业务收入，估算出 SAP 软件业务的价值。我对咨询业务也做了同样的事情。根据我的估算，SAP 大约 80% 是软件业务，20% 是咨询业务。我用这个权重对之前得到的两个行业的无杠杆 Beta 进行加权平均，得到了 SAP 作为一个业务组合的无杠杆 Beta。

最后一步，我查找了 SAP 的负债权益比。SAP 并不是一家大量使用债务的公司，其负债权益比仅为 1.4%。利用这个负债权益比和德国的公司税率，我将之前计算出的无杠杆 Beta 重新杠杆化，最终得到了 SAP 公司的股权 Beta。这就是我最终会在估值中使用的 Beta，而不是回归 Beta。

总结一下，你需要一个相对风险的衡量标准。你可能不喜欢 Beta，没关系，但不要因此就放弃衡量相对风险。你可以使用基于会计指标或定性因素的替代方案。我之所以需要一个相对风险的度量，是因为如果没有它，我将对风险状况截然不同的公司使用相同的贴现率。所以，不要让你对 Beta 的不屑导致你走上这条错误的道路。找到一个你认可的相对风险衡量方法，我们今天讨论的所有逻辑和步骤依然适用。
